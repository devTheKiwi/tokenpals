# 04. UI & 캐릭터 시스템

## UI 구조 개요

```
┌─ 메뉴바 (항상 보임, 보조) ─┐
│   🥔 67%                    │   ← 가장 활발한 디바이스 + 5h 진행률
└────────────────────────────┘
            ↓ 클릭
       (작은 메뉴: 방 열기, 요약, 설정, 종료)

──── 핵심 UX ────

┌─ 방 윈도우 (메인 화면, 핀 가능) ────────┐
│                                          │
│   여러 디바이스 캐릭터들이 한 방 안에     │
│   살아서 돌아다니거나, 앉거나, 자거나    │
│   상태에 따라 다른 행동을 함              │
│                                          │
└──────────────────────────────────────────┘

별도:
- 데스크톱 위젯 (Phase 1.5)
- 알림 (한도 임박, 업적 해금)
- 설정 윈도우
```

## 메뉴바 아이콘 (보조)

상황별 아이콘 변화:

| 상태 | 표시 | 비고 |
|---|---|---|
| 전부 유휴 | `🥔💤` | 가장 캐릭터 + 자는 표시 |
| 활성 작업중 | `🥔 67%` | 가장 활발한 디바이스 + 5h 진행률 |
| 한도 임박 | `🥔 92%⚠️` | 빨간 경고 색 |
| 오프라인 | `🥔 ⚪` | 흐릿한 아이콘 |
| 멀티 계정 (Phase 2) | `🥔+2` | 메인 + 다른 계정 N개 |

**기술**: `NSStatusItem` + 동적 갱신. 클릭시 작은 메뉴 (방 열기/숨기기, 빠른 요약, 설정, 종료).

## 메인 UX — 방 윈도우 (Room Window)

핀 가능한 작은 윈도우. 디바이스마다 한 마리씩의 캐릭터가 한 방에 모여 살아 있음.

### 기본 레이아웃 (Phase 1: 빈 방)

```
┌─ 🐾 TokenPals ──────────[📌][−][×]┐
│  내 가족 (3마리)  · 5h 67%  · 주간 31%│
├──────────────────────────────────┤
│                                    │
│                                    │
│   🥔─→  "코딩중!"                  │  ← 활동중인 캐릭터, 작은 말풍선
│                                    │
│                                    │
│              🥔💤                   │  ← 가만히 있거나 졸린 캐릭터
│                                    │
│                                    │
│   🥔 ⚫  (그림자만)                  │  ← 오프라인 캐릭터
│                                    │
│                                    │
├──────────────────────────────────┤
│  💰 279K  🔋 67%  📊 통계  ⚙️       │
└──────────────────────────────────┘
```

Phase 1에서는 **빈 방**으로 시작 (가구·장식 없음). 캐릭터가 자유롭게 위치하고 움직이는 것만 구현. 방 꾸미기·가구·테마는 Phase 3로.

### 윈도우 사양

| 항목 | 값 |
|---|---|
| **기본 크기** | 480 × 360 pt |
| **최소 크기** | 320 × 240 pt |
| **최대 크기** | 화면 절반 (디바이스 많을 때 대응) |
| **리사이즈** | 가능 (모서리 드래그) |
| **이동** | 가능 (배경 드래그 또는 타이틀바) |
| **핀 (Always on top)** | 토글 가능 (📌 버튼) |
| **다중 데스크톱** | 핀시 모든 Space에서 보임 |
| **상단 바** | 디바이스 수 + 5h% + 주간% (요약) |
| **하단 바** | 합계 토큰 + 핵심 지표 + 진입점 (통계/설정) |

### 핀 동작

```swift
// 핀 ON
window.level = .floating
window.collectionBehavior = [.canJoinAllSpaces, .stationary]

// 핀 OFF
window.level = .normal
window.collectionBehavior = .default
```

핀 켜면: 다른 앱·창 위에 항상, 모든 데스크톱에서 보임. 작업 중에도 곁눈질 가능.

## 캐릭터 시스템

### 디자인 베이스 (ClaudePet 차용)

ClaudePet의 **감자형 캐릭터**(NSBezierPath 직접 드로잉)를 그대로 가져옴. 핵심 요소:
- **몸통**: 둥근 사각형 (40×30, 반경 14)
- **머리**: 둥근 사각형 (32×18, 반경 12)
- **발**: 둥근 사각형 두 개 (8×10)
- **눈**: 흰자 + 동공 (동공이 이동방향 따라 움직임)
- **6색 팔레트**: 오렌지 / 블루 / 그린 / 퍼플 / 핑크 / 틸 (PetColor.palette)

이 패턴 그대로 활용. 디바이스마다 색상이 달라서 구분.

### 디바이스 = 캐릭터 매핑

| 단위 | 캐릭터 | 비고 |
|---|---|---|
| 디바이스 1대 | 캐릭터 1마리 | 디바이스 등록시 색상 선택 |
| 디바이스 이름 | 캐릭터 위 작은 라벨 | "맥북프로" 등 |
| 활성 Claude 세션 | 작업 이펙트 (반짝임) | ClaudePet의 working effect 차용 |
| Claude Desktop 실행 | 부엌존에서 ☕ 들고 있음 | ClaudePet의 desktop mode 차용 |

(ClaudePet은 "세션마다 캐릭터"였지만, TokenPals는 "디바이스마다 캐릭터")

## 상태 → 방 안 행동 매핑

캐릭터의 위치/행동이 정보 표시 그 자체:

| Mood | 방 안 행동 | 추가 표시 |
|---|---|---|
| `working` | 한 자리에서 가만히 + 반짝임 이펙트 (몰입) | 작은 말풍선 "코딩중!" |
| `happy` | 방 안 천천히 좌우로 산책 | 가끔 멈춰서 깜빡 |
| `sleepy` | 한 곳으로 가서 정지 + 💤 | 30분+ 유휴시 |
| `confused` | 갸웃거리며 머리 긁는 모션 | 말풍선 "/clear 어때?" |
| `stressed` | 한 자리에서 작은 진동 (안절부절) | 말풍선 "토큰 많이 먹었어..." |
| `danger` | 빨간 글로우 + 점프 | 말풍선 "잠깐만 쉬자!!" |
| `offline` | 그림자만 (반투명, 위치 고정) | 라벨도 흐림 |

### 상태 결정 로직

```swift
enum Mood { case working, happy, sleepy, confused, stressed, danger, offline }

func computeMood(_ device: Device) -> Mood {
    guard device.isOnline else { return .offline }
    if device.fiveHourUsage > 0.90 { return .danger }
    if device.fiveHourUsage > 0.75 { return .stressed }
    if device.currentSessionActive { return .working }
    if device.cacheHitRate < 0.3 && device.currentTokens > 50_000 { return .confused }
    if device.minutesSinceLastActivity > 30 { return .sleepy }
    return .happy
}
```

### 캐릭터 행동 / 위치 시스템

방은 좌표계가 있고, 캐릭터는 그 안에서 자유롭게 움직임:

```swift
class PetActor {
    var position: CGPoint               // 방 안 좌표
    var targetPosition: CGPoint?         // 가야 할 곳
    var currentBehavior: Behavior        // .standing | .walking(dir) | .sleeping | ...
    var color: PetColor                  // 디바이스별 색
    var deviceId: UUID

    enum Behavior {
        case standing
        case walking(left: Bool)
        case jumping
        case sleeping
        case fidgeting   // 안절부절
    }

    func update(deltaTime: TimeInterval, roomBounds: CGRect) {
        // 매 프레임 위치/상태 갱신
        // ClaudePet PetView의 tick() 로직 차용
    }
}
```

ClaudePet의 PetView 코드를 거의 그대로 차용 — 단, 좌표계가 "화면 전체"가 아니라 "방 영역" 안으로 제한됨.

### 말풍선 (SpeechBubble)

ClaudePet의 SpeechBubbleWindow 패턴 차용 — 단, 별도 NSWindow가 아닌 **방 윈도우 내부 NSView**로 변경:
- 캐릭터 머리 위에 작은 말풍선 표시
- 자동 페이드 (3~5초 후)
- 일정 확률로 랜덤 발화 (45~90초마다, ClaudePet 차용)
- 클릭하면 다음 멘트로 전환

## 방 안 인터랙션

| 액션 | 결과 |

## 방 테마 / 꾸미기 (Phase 3)

Phase 1은 빈 방으로 시작. 이후 단계적 확장:

- **Phase 1**: 빈 방, 좌표 기반 자유 이동
- **Phase 1.5**: 시간대별 조명 (새벽 어둠, 오후 햇살) — 배경 그라데이션만
- **Phase 3 (선택)**: 가구·존(zone) 시스템 도입
  - 책상 (working zone): working 캐릭터가 모임
  - 침대 (sleeping zone): sleepy 캐릭터가 누움
  - 부엌 (coffee zone): Claude Desktop 켜진 캐릭터가 ☕와 함께
  - 창문 (window): 시간대별 풍경
- **Phase 3+**: 사용 토큰으로 가구 구매 (다마고치 + 동물의 숲)
- **Phase 3+**: 계절 테마 (봄 에디션 — ClaudePet 차용)

## 기술 스택 / 패턴

| 항목 | 기술 | ClaudePet 차용 여부 |
|---|---|---|
| 윈도우 | `NSWindow` (커스텀 borderless or titled) | 패턴 차용 |
| 캐릭터 드로잉 | `NSView` + `NSBezierPath` + `Timer 15fps` | ✅ 그대로 |
| 캐릭터 색상 팔레트 | 6색 (PetColor) | ✅ 그대로 |
| 상태 머신 | enum `PetState` | ✅ 패턴 |
| 말풍선 | NSView 내부 미니 컴포넌트 | 패턴 차용 (windowless) |
| 메뉴바 | `NSStatusItem` + `NSMenu` | ✅ 패턴 |
| 핀 토글 | `window.level = .floating` | 신규 |
| 윈도우 리사이즈 | `styleMask`에 `.resizable` | 신규 |
| 알림 | `UserNotifications` | 신규 |
| 위젯 | `WidgetKit` | 신규 |

## 디자인 가이드라인

- **모서리 반경**: 12pt (방 윈도우), 8pt (작은 요소)
- **간격**: macOS HIG 8pt 그리드
- **타이포그래피**: SF Pro (system), 강조에 SF Pro Rounded
- **색상**: macOS Dynamic Colors (다크/라이트 자동)
- **방 배경**: Phase 1은 단색 (다크모드 자동 대응)
- **여백**: 답답하지 않게, 캐릭터 주변 최소 20pt

## 접근성

- VoiceOver로 캐릭터 상태 읽기 ("맥북프로, 활성, 67% 사용중")
- Dynamic Type 지원
- 색상에만 의존 X (텍스트 라벨 함께)
- 캐릭터 움직임 줄이기 옵션 (모션 감도 사용자에게)

## ClaudePet과의 공존

```
[데스크톱 전체]
  ├─ ClaudePet 펫 (자유롭게 떠다님, Claude 세션 단위)
  └─ TokenPals 방 윈도우 (작은 창, 디바이스별 캐릭터들이 안에서만 활동)
```

서로 영역 침범 X. 의미도 보완:
- ClaudePet = "지금 이 머신의 Claude 세션이 살아있다"
- TokenPals = "내 모든 머신의 디바이스 상태"

## 미정 디자인 결정

- [ ] 캐릭터를 ClaudePet과 동일하게 (감자형) vs 살짝 변형 (귀모양 추가 등)으로 차별화
- [ ] 말풍선 멘트 톤 — 귀여움 / 차분함 / 유머 비율
- [ ] 알림 빈도 — 너무 잦으면 끔, 적으면 효용 떨어짐
- [ ] 캐릭터 이동 속도 (너무 빠르면 산만, 너무 느리면 정지처럼 보임)
- [ ] 방 배경 색상 (단색 vs 미세한 패턴)
- [ ] Phase 3에서 도입할 zone 시스템의 첫 가구 구성
