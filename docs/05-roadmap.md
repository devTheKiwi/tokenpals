# 05. 로드맵

## 마일스톤 요약

| Phase | 기간 (추정) | 목표 | 결과물 |
|---|---|---|---|
| **0. 셋업** | 1~2일 | 개발 환경, Supabase 프로젝트 | 빈 메뉴바 앱 + DB 연결 |
| **1. 단일 계정 MVP** | 2주 | 멀티 디바이스, 캐릭터, 기본 모니터링 | 본인이 매일 쓰는 수준 |
| **1.5. 다듬기** | 1주 | Realtime, 알림, 위젯, 애니메이션 | 자랑하고 싶은 수준 |
| **2. 멀티 계정 / 팀** | 1주 | 동료 초대, 팀 뷰 | 동료에게 풀 수 있는 상태 |
| **3. 분석/지능형 (선택)** | 1~2주 | 패턴 분석, 자동 제안, 업적 시스템 | 토큰 진짜 절약되는 단계 |

총 **4~6주** 풀타임 기준. 부분 시간이면 2~3개월 예상.

## Phase 0: 개발 환경 셋업 (1~2일)

### 작업
- [ ] **Xcode 프로젝트 생성** (macOS App, SwiftUI)
- [ ] **메뉴바 앱 보일러플레이트** (`NSStatusItem` + `NSPopover`)
- [ ] **Supabase 프로젝트 생성** (무료 티어)
- [ ] **Supabase Swift SDK 통합** (Swift Package)
- [ ] **Google OAuth 설정** (Supabase Auth)
- [ ] **로그인 화면** (간단한 버튼 + 콜백)
- [ ] **로컬 SQLite 셋업** (`GRDB` 또는 `SQLite.swift`)
- [ ] **기본 스키마 마이그레이션** (Supabase + 로컬 동시)

### 종료 조건
- 앱 실행 → 로그인 → 빈 팝오버 표시
- Supabase에 로그인 사용자 기록 확인

## Phase 1: 단일 계정 MVP (2주)

### 1주차: 데이터 파이프라인

- [ ] **JSONL 파일 워처 구현**
  - `~/.claude*/projects/**/*.jsonl` 패턴 탐색
  - `FSEvents` (or `DispatchSource`) 로 변경 감지
  - 마지막 offset 기억 (`jsonl_cursors` 테이블)
- [ ] **JSONL 파서**
  - 라인 단위 파싱 (`Codable`)
  - 모델, 토큰, 도구, 타임스탬프 추출
  - 새 라인만 처리 (idempotent)
- [ ] **로컬 DB 적재**
  - sessions, turns 테이블 채우기
  - UNIQUE 제약으로 중복 방지
- [ ] **백필 (기존 JSONL 전체 흡수)**
  - 첫 실행시 진행률 표시
  - 배치 INSERT
- [ ] **Supabase 동기화**
  - 동기화 큐 → 백그라운드 워커
  - 네트워크 끊김시 재시도

### 2주차: UI & 캐릭터

- [ ] **디바이스 등록 플로우**
  - 첫 실행: 이름 + 캐릭터 선택
  - hostname 자동 추천
  - Keychain에 device_id 저장
- [ ] **메뉴바 아이콘 동적 갱신**
  - 가장 활발한 디바이스 캐릭터 표시
  - 5h 진행률 % 같이
- [ ] **팝오버 메인 뷰**
  - 디바이스 카드 리스트
  - 합계 영역 (5h, 주간)
- [ ] **캐릭터 상태 머신**
  - mood 계산 함수
  - 표정/말풍선 매핑
- [ ] **계정 디렉토리 자동 감지**
  - `.claude*` 패턴 스캔
  - 사용자에게 매핑 확인 요청

### 종료 조건 (Phase 1)
- 멀티 디바이스에서 앱 설치 → 자동 동기화
- 한 디바이스 메뉴바 클릭 → 다른 디바이스 사용량까지 보임
- 캐릭터 상태가 실제 사용 패턴 따라 변화

## Phase 1.5: 다듬기 (1주)

- [ ] **Supabase Realtime 구독**
  - 다른 디바이스 새 turn → 즉시 UI 갱신
- [ ] **알림 (UserNotifications framework)**
  - 5h 80% / 95% 임계치
  - 알림 설정 화면
- [ ] **데스크톱 위젯 (WidgetKit)**
  - 작은/중간 사이즈
  - App Group으로 메인 앱과 데이터 공유
- [ ] **애니메이션**
  - 숨쉬기, 깜빡, mood 전환
- [ ] **상세 뷰**
  - 카드 클릭 → 시간대별 그래프
  - 최근 세션 리스트
  - SwiftUI Charts 활용
- [ ] **설정 화면**
  - 알림 토글
  - 디바이스/캐릭터 변경
  - 데이터 내보내기

### 종료 조건 (1.5)
- 매일 켜놓고 봐도 부담스럽지 않음
- 한도 임박시 알림으로 행동 변화 가능
- 동료가 봤을 때 "오 이거 뭐야?" 반응

## Phase 2: 멀티 계정 / 팀 (1주)

- [ ] **계정 데이터 모델 분리**
  - `account_id` 모든 쿼리에 적용
  - RLS 정책 활성화
- [ ] **팀 초대 흐름**
  - 사용자 A: "동료에게 보기 권한 공유" 버튼
  - 초대 링크/코드 생성
  - 동료가 자기 앱에서 수락 → `account_links` INSERT
- [ ] **다중 계정 UI**
  - 계정 탭 또는 색상 구분
  - "내 가족 vs 친구 가족" 시각적 분리
- [ ] **권한 관리**
  - viewer는 read-only
  - 자기 계정 데이터만 write
- [ ] **팀 합산 대시보드**
  - 전체 팀 5h 사용량 시각화
  - 누가 한도 임박인지 한눈에

### 종료 조건 (Phase 2)
- 동료에게 dmg 공유 → 동료가 설치 → 본인 계정 본인이 보고 → 너 계정도 너가 본인 모니터링하면서 동료 계정 함께 보기 가능

## Phase 3: 분석/지능형 (선택, 1~2주)

- [ ] **사용 패턴 분석**
  - 시간대별 평균 burn rate
  - 프로젝트별 토큰 효율
  - 비싼 세션 TOP 10
- [ ] **자동 제안 시스템**
  - "이 패턴이면 한도 X시간 후 도달"
  - "/compact 또는 /clear 권장 타이밍"
  - "Opus 대신 Sonnet 추천"
- [ ] **업적 시스템**
  - 캐시 마스터, 효율왕, 야근러 등
  - 해금 알림 + 카드 뷰에 배지
- [ ] **연속 사용 streak**
  - 디바이스별 streak 카운트
  - 1주년 / 100세션 등 마일스톤
- [ ] **Lottie 애니메이션 도입**
  - 정적 이모지 → 살아있는 캐릭터

## Phase 4+: 미래 가능성 (메모)

- iOS/iPadOS 컴패니언 앱 (외부에서 한도 확인용)
- Slack 연동 (한도 임박시 본인 채널에 알림)
- 멀티 인공지능 (Claude / Codex / Cursor) 통합
- 공개 앱스토어 출시
- 팀 단위 비용 분석 (예산 관리)

## 일정 추정 가정

- 1주 = 풀타임 5일 기준 (또는 부업 10~15시간/주의 2배 시간)
- Swift/SwiftUI 학습 곡선 별도 (기존 경험 무 → +1주)
- 디자인/에셋 작업 별도 (이모지 베이스면 0)

## 위험 요소

| 위험 | 영향 | 대응 |
|---|---|---|
| Claude Code JSONL 포맷 변경 | 파싱 깨짐 | 버전 감지 + fallback 파서 |
| Supabase 무료 한도 초과 | 동료 늘면 발생 가능 | 압축/요약 정책, 유료 플랜 검토 |
| Apple 코드 사이닝 / 공증 | 배포 마찰 | Phase 2 전에 Apple Dev 가입 검토 |

| 멀티 `.claude*` 폴더 스키마 차이 | 파싱 실패 | 폴더별 독립 파서 옵션 |
