# 01. 프로젝트 개요

## 목표

Claude Code Max 구독 사용자가 **여러 디바이스(Phase 1)와 여러 계정(Phase 2)의 토큰 사용량을 통합적으로 모니터링**하고, 사용 패턴을 분석해서 토큰을 아낄 수 있게 돕는다.

## 핵심 가치 제안

1. **통합 모니터링**: Claude Code를 여러 머신에서 써도 한 화면에서 합산
2. **실시간**: 다른 디바이스에서 쓰는 중에도 1초 이하로 반영
3. **예측·경고**: 5h 윈도우·주간 한도 임박시 알림
4. **방 안의 픽셀 공 가족**: 핀 가능한 작은 방 윈도우에 픽셀 공 캐릭터가 살아 움직임. 상태에 따라 위치·행동이 변함 (다마고치 + 사무실 컨셉)
5. **로컬 우선**: 프롬프트 본문은 절대 외부 전송 X, 메타데이터만

## 핵심 컨셉: 방 안의 픽셀 공 가족

```
┌─ TokenPals (핀 가능, 480×360 리사이즈) [📌][−][×]┐
│  성랑의 맥 · 오늘 234K · 5시간 11% · 캐시 96%      │
├────────────────────────────────────────────────┤
│         (벽 — 위쪽 절반, 크림색)                 │
│  ─────────────────────────────────────────       │
│      🥔 ✨  "코딩중!"                            │ ← working: 머리 위 스파클
│         (바닥 — 아래쪽 절반, 픽셀 도트 패턴)      │
├────────────────────────────────────────────────┤
│  💰 234K   📊 통계   ⚙️ 설정                       │
└────────────────────────────────────────────────┘
```

**캐릭터 모델**: 12×12 픽셀 공 (아트 스타일은 [04-ui-and-characters.md](04-ui-and-characters.md))
- ㅅ 모양 작은 입 + 핑크 볼터치 + 클릭하면 ^^ 웃음
- 6색 팔레트 (디바이스별 색상)

**진행 현황 (현재 Phase 2 진행중)**: **1 Claude 계정 = 1 펫 가족 (= 1 방)**
- **Phase 1 완료**: 1 계정 + 1 디바이스 (이 머신)
- **Phase 2 진행중**: Email OTP 로그인 + 디바이스 등록 구현 완료. 다음은 Realtime 동기화
  - 1 계정 + 2+ 디바이스 (멀티 디바이스 sync 예정)
- **Phase 3+**: 2+ 계정 (다른 방으로 분리), 동료 초대

**Identity 모델**: TokenPals 로그인(Email OTP via Supabase Auth)이 사용자 식별. 같은 로그인 = 같은 사람. Claude 계정 자동 감지는 안 함 (보안/안정성). Phase 2에서 비밀번호 없는 간편 로그인으로 구현 완료.

## 타겟 사용자 / 진화 경로

### Phase 1: 단일 계정, 단일 디바이스 (현재)
- 1 Claude 계정 (`~/.claude/`)
- 1 머신 (이 Mac)
- **본인의 토큰 절약 도구**로 충분히 동작

### Phase 2: 단일 계정, 멀티 디바이스
- 같은 사람의 맥북/데스크탑/노트북 통합
- Supabase 동기화로 디바이스간 합산
- 한 계정 = 한 방, 펫 여러 마리 (디바이스마다 1마리)

### Phase 3+: 멀티 계정 (사용자의 본인 다른 계정 + 동료 계정)
- 본인 다른 Claude 계정도 추가 (`.claude-alt` 등)
- 동료 계정 초대 (보기 권한)
- 계정별로 별도 방 (또는 색상 구분)

## 범위 (Scope)

### 포함
- ✅ Claude Code (CLI) 사용량 추적
- ✅ macOS 네이티브 메뉴바 앱
- ✅ 멀티 디바이스 합산 (단일 계정)
- ✅ 멀티 계정 합산 (팀 모드)
- ✅ 5h 윈도우·주간 한도 추정
- ✅ 캐시 효율 분석
- ✅ Opus/Sonnet/Haiku 모델별 분리

### 제외
- ❌ **claude.ai 웹** 사용량 (로컬에 데이터 없음 — 본질적 한계)
- ❌ **모바일 앱** 사용량 (동일 이유)
- ❌ Anthropic API 키 직접 사용량 (다른 도구 영역)
- ❌ Windows / Linux 클라이언트 (필요해지면 별도 검토)

## 비기능 요구사항

| 항목 | 목표 |
|---|---|
| 방 윈도우 응답성 | 표시 → 첫 프레임 100ms 이내 |
| 캐릭터 애니메이션 | 15fps 이상, 부드러움 |
| 실시간 동기화 지연 | 1초 이하 |
| 앱 사이즈 | 10MB 이하 |
| 메모리 사용 | 50MB 이하 (idle), 200MB 이하 (활성) |
| 배터리 영향 | macOS 권장 best practice 준수 |
| 프라이버시 | 프롬프트 본문 비전송, 메타데이터만 |

## 성공 지표 (스스로 평가)

- 매일 메뉴바 클릭하게 되는가? (사용 빈도)
- 한도 도달 빈도가 줄었는가? (행동 변화)
- 동료에게 자랑하고 싶어지는가? (제품 완성도)
- 동료가 자발적으로 깔고 싶어하는가? (확산성)

## 컨셉의 영감

- **다마고치**: 각 디바이스가 살아있는 생물처럼 느껴지게
- **동물의 숲 / 작은 사무실**: 한 공간에 여러 캐릭터가 모여서 각자 활동
- **ClaudePet** ([github](https://github.com/devTheKiwi/ClaudePet)): 같은 사용자의 선행 프로젝트. 캐릭터 드로잉/애니메이션 패턴 차용
- **TokenTracker** ([github](https://github.com/mm7894215/TokenTracker)): 메뉴바 + 위젯 패턴
- **Claude-Usage-Tracker** ([github](https://github.com/hamed-elfayome/Claude-Usage-Tracker)): Swift 네이티브 사례

## ClaudePet과의 관계 / 차별점

[ClaudePet](https://github.com/devTheKiwi/ClaudePet)은 동일 사용자(devTheKiwi)의 선행 프로젝트로, 컨셉/구현 패턴을 적극 차용한다.

| 항목 | ClaudePet | TokenPals |
|---|---|---|
| **컨셉** | 데스크톱 위를 떠다니는 펫 | **핀 가능한 방 윈도우** 안에 픽셀 공이 살아 움직임 |
| **캐릭터 단위** | Claude 세션마다 펫 1마리 | **디바이스마다 캐릭터 1마리** (계정 컨텍스트 안에서) |
| **시각 스타일** | 벡터 (NSBezierPath 둥근 형태) | **픽셀 아트** (12×12 공, 안티앨리어싱 OFF) |
| **표시 영역** | 데스크톱 전체 | 자기 방 윈도우 안 (충돌 X) |
| **합산 시각화** | 없음 (단일 머신) | **여러 디바이스/계정 통합** (Phase 2+) |
| **백엔드** | 없음 (로컬 only) | **Supabase 연동, Realtime sync** (Phase 2+) |
| **관계** | 별도 프로젝트로 계속 운영 | ClaudePet 패턴 차용한 진화형 |

**공존 가능**: 데스크톱에서 ClaudePet 펫이 돌아다니고, TokenPals 방 윈도우에 다른 캐릭터들이 모여 있는 구도. 의미가 보완적이라 동시 사용도 자연스러움.

## 정식 명칭

**TokenPals** (토큰팔즈) — 결정 완료.

선정 근거:
- 상표 안전 (Anthropic "Claude" 상표권 회피) → App Store 배포·정식 유통에 마찰 없음
- 확장성 — 향후 Codex/Cursor/Gemini 등 다른 AI 도구 통합해도 이름 유효
- "토큰 친구들이 지켜준다"는 컨셉을 짧게 표현
- 부제로 도메인 명확화: *TokenPals — Claude Code 토큰을 귀여운 친구들이 지켜드려요*
